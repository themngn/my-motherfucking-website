Simplicity is the key

There is little point in making a website look good if it's not going to be used.
I made this website for myself, so i don't care about how it looks. But i care if i will use it. I have a lot of unfinished projects. And this will not be one of them.
It is already finished. I already using it. Yes, i will update it with more stuff. But i don't need to update it with styles or new features to make it usable, because it already is.
Also, benefits of simplicity are:

  • It's fast
  • Responsive
  • Backward compatible
  • It's easy to maintain
  • Free hosting on github
  • I don't need to make backups because i use git. So basically every time i edit something i make a backup.
